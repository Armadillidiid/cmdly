# Node SEA Cutover Plan

## Goal
Replace `ncc` packaging with Node SEA. Full cutover now. Primary distribution: GitHub Release binaries only. No Windows in v1. Unsiged binaries in v1.

## Confirmed Decisions
- Distribution: GitHub binaries only.
- Cutover: full cutover now (remove npm JS bin flow).
- Targets v1: Linux + macOS ARM64, exclude Windows.
- Intel Mac: exclude `darwin-x64` for now (known upstream SEA risk).
- Runtime pin: latest Node 26.x for SEA build.
- Signing: none in v1.

## Implementation Steps
1. Update build + package metadata (`package.json`)
- Remove `ncc` build flow and `postbuild` dependency chain.
- Remove npm `bin` mapping to `dist/index.js` and `files: ["dist"]` packaging assumptions.
- Add SEA-focused scripts, example shape:
  - `build:bundle` -> produce single JS entry artifact for SEA input.
  - `build:sea` -> generate SEA executables for `linux-x64` and `darwin-arm64` with pinned Node 26.x.
  - `build` -> run SEA build pipeline.
  - `verify:sea` -> smoke-run each built binary.
- Remove `@vercel/ncc` devDependency.

2. Add SEA build orchestrator + config
- Add `scripts/build-sea.mjs` to:
  - create temporary SEA config JSON per target,
  - call Node SEA generation (`node --build-sea ...`),
  - emit binaries into deterministic output dir (example `dist-app/<target>/cmdly`).
- Use `mainFormat: "module"` if bundle stays ESM; if bundler emits CJS, set `mainFormat: "commonjs"`.
- Set `useCodeCache: false` for cross-target safety.
- Set `useSnapshot: false` in v1.
- Keep runtime FS behavior unchanged (config/credential/cache paths already under user home).

3. Replace constant injection approach
- Remove dependency on `scripts/inject-constant.js` mutating built output.
- Inject app name/version during bundle generation step (define/env/templating), so SEA input file already finalized.
- Verify `src/constants.ts` still resolves placeholders correctly in non-release/dev mode.

4. Update release workflow (`.github/workflows/release.yml`)
- Keep changesets versioning/publish logic only if still needed for tags/changelog.
- Replace release build step with SEA build steps.
- Add matrix or explicit jobs for:
  - `linux-x64` build on Ubuntu,
  - `darwin-arm64` build on macOS runner.
- Upload binaries + checksums as GitHub Release assets.
- Ensure release notes include target list and install instructions.

5. Update docs (`README.md`)
- Remove npm global install as primary install path.
- Add binary install/usage instructions by platform:
  - download from GitHub Releases,
  - `chmod +x` on Unix,
  - add to `PATH` or move to user bin dir.
- Add known limitations section:
  - no Windows support in v1,
  - no Intel Mac binary in v1.

6. Cleanup obsolete paths
- Delete `scripts/inject-constant.js` after replacement done.
- Remove any `dist/` assumptions in scripts/docs.
- Keep local dev command (`node --import=tsx src/bin.ts --watch`) unchanged.

## Critical Files To Modify
- `package.json`
- `.github/workflows/release.yml`
- `README.md`
- `src/constants.ts`
- `scripts/build-sea.mjs` (new)
- `scripts/inject-constant.js` (delete)

## Verification Plan
1. Build artifacts
- Run `pnpm build`.
- Confirm outputs exist for:
  - `dist-app/linux-x64/cmdly`
  - `dist-app/darwin-arm64/cmdly`

2. Binary smoke tests
- `./dist-app/linux-x64/cmdly --help`
- `./dist-app/linux-x64/cmdly --version`
- Run representative command path that triggers:
  - model fetch/cache (`src/utils/models.ts` path),
  - action handling (`run`/`copy`/`explain` from `src/utils/actions.ts`).

3. Runtime filesystem behavior
- Execute with isolated HOME (`HOME=$(mktemp -d)`) and verify config + credential files create in user dirs.

4. CI/release validation
- On main push, ensure release workflow builds both targets and attaches binaries/checksums.
- Validate download-and-run from generated GitHub Release assets.

## Risks + Mitigations
- ESM/CJS SEA entry mismatch -> lock bundle output format and set matching `mainFormat`.
- Cross-target instability with code cache/snapshot -> keep both disabled in v1.
- macOS ARM build requirements -> run mac target build on macOS runner.
- Distribution breakage from full cutover -> add pre-release dry run before first public release.

## Rollback
- Reintroduce previous JS packaging scripts (`ncc` + `dist/index.js` bin) via single revert commit.
- Keep release tags/changesets flow independent so binary pipeline failure does not block code release hotfix.
